  1. WHAT IS MARKETNOISE?
────────────────────────────────────────────────────────────────────────────────

MarketNoise is a narrative dynamics platform for retail investors. The core
mission is NOT to predict stock prices — it is to help users understand the
STORY being told about a stock: whether it is hype-driven or information-driven,
whether the narrative is accelerating or cooling, and when the story shifted.

The product answers three questions:
  • What is the market saying about this stock right now? (sentiment)
  • Is the attention growing or fading? (velocity)
  • Is this information-driven or hype-driven? (hype score)

And two deeper questions:
  • How has the narrative evolved over time? (narrative drift)
  • What archetype is this stock in right now? (pattern classification)

  5. DATABASE MODELS
────────────────────────────────────────────────────────────────────────────────

  Stock
  ├── ticker          CharField (unique, indexed)
  ├── name            CharField
  ├── sector          CharField
  ├── exchange        CharField
  └── is_active       BooleanField

  Narrative  (a news article or social post about a stock)
  ├── stock           FK → Stock
  ├── title           CharField(500)
  ├── content         TextField
  ├── source          CharField (news/rss/reddit/twitter/other)
  ├── source_name     CharField
  ├── url             URLField (unique constraint on non-empty)
  └── published_at    DateTimeField

  SentimentScore  (FinBERT/VADER result per Narrative)
  ├── narrative       OneToOneField → Narrative
  ├── stock           FK → Stock
  ├── label           CharField (positive/negative/neutral/mixed)
  ├── positive_score  FloatField
  ├── negative_score  FloatField
  ├── neutral_score   FloatField
  └── compound_score  FloatField  (-1 to +1)

  VelocityMetric  (mention rate change over a time window)
  ├── stock           FK → Stock
  ├── mention_count   IntegerField (total articles for this stock)
  ├── velocity_score  FloatField  (0–100, rate of change)
  ├── acceleration    FloatField  (% change vs prior period)
  ├── trend           CharField (accelerating/decelerating/stable)
  ├── window_start    DateTimeField
  └── window_end      DateTimeField

  HypeScore  (composite risk score per analysis run)
  ├── stock               FK → Stock
  ├── score               FloatField  (0–100)
  ├── level               CharField (low/moderate/high/extreme)
  ├── sentiment_imbalance FloatField  (raw imbalance ratio 0–1)
  ├── velocity_factor     FloatField  (raw velocity score 0–100)
  ├── social_news_ratio   FloatField  (reserved, currently 0)
  └── explanation         TextField

  PriceImpact  (tracks price movement after a narrative, for ML training)
  ├── narrative       OneToOneField → Narrative
  ├── stock           FK → Stock
  ├── price_at_publish / price_1h / price_4h / price_24h  FloatField
  ├── impact_1h / impact_4h / impact_24h  FloatField (% change)
  └── direction_24h   CharField (up/down/flat)

  UserAPIKey  (encrypted storage of user's LLM keys)
  ├── user            FK → auth.User
  ├── service         CharField (gemini/groq)
  └── encrypted_key   TextField (Fernet-encrypted)

  6. DATA FLOW — WHAT HAPPENS WHEN YOU SEARCH A TICKER
────────────────────────────────────────────────────────────────────────────────

  User searches "NVDA"
        │
        ▼
  GET /api/market/analyze/NVDA/?name=NVIDIA
        │
        ├─▶ Stock.get_or_create(ticker='NVDA')
        │
        ├─▶ Check Narrative DB for articles in last 48h
        │     └─ If empty → fetch from GNews API (fallback)
        │          └─ Save new Narrative records to DB
        │
        ├─▶ ensure_sentiment_scores()
        │     └─ Find Narratives with no SentimentScore
        │          ├─ Batch all titles through FinBERT (or VADER fallback)
        │          └─ Save SentimentScore records to DB
        │
        ├─▶ compute_velocity()
        │     ├─ Count narratives: last 24h vs 24-48h ago
        │     ├─ Compute % change → velocity score 0-100
        │     └─ Save VelocityMetric to DB
        │
        ├─▶ compute_hype_score()
        │     ├─ Factor 1: sentiment imbalance (0-40 pts)
        │     ├─ Factor 2: velocity factor (0-35 pts)
        │     ├─ Factor 3: source concentration (0-25 pts)
        │     └─ Save HypeScore to DB
        │
        ├─▶ classify_pattern(sentiment, velocity, hype)
        │     └─ Rule-based classifier → one of 6 archetypes
        │
        └─▶ Return JSON { sentiment, velocity, hype, pattern }
              └─ Cached for 15 min

  Separately:
  GET /api/market/drift/NVDA/
        │
        ├─▶ Query all SentimentScore records for NVDA (last 91 days)
        ├─▶ Aggregate by day → daily timeline [{date, compound, count, label}]
        ├─▶ Compute window stats: 7d / 30d / 90d
        │     └─ Compare first half vs second half → drift direction
        ├─▶ Detect last narrative shift (3-day rolling average crossing)
        └─▶ Return { ticker, timeline, windows, shift, data_available }
              └─ Cached for 30 min

  8. NLP PIPELINE — FINBERT + VADER
────────────────────────────────────────────────────────────────────────────────

  Model: ProsusAI/finbert
  ├── Trained on financial text (Reuters, Financial PhraseBank)
  ├── Outputs: positive_prob, negative_prob, neutral_prob
  └── compound = positive_prob - negative_prob  (range -1 to +1)

  Singleton pattern (api/nlp/finbert.py):
  ├── Lazy-loaded on first call (avoids startup memory hit)
  ├── Device detection: MPS (Apple Silicon) > CUDA > CPU
  ├── Batch inference: analyze_batch(texts, batch_size=32)
  └── is_available() → False if model not downloaded yet

  Fallback (NLTK VADER):
  ├── Always available, no download required beyond lexicon
  ├── Not trained on financial text → weaker signal
  └── Used when FinBERT is not installed or fails

  Classification thresholds (compound score):
  ├── compound >= +0.05  → positive (bullish)
  ├── compound <= -0.05  → negative (bearish)
  └── else               → neutral


────────────────────────────────────────────────────────────────────────────────
  9. PATTERN CLASSIFICATION (6 ARCHETYPES)
────────────────────────────────────────────────────────────────────────────────

  Classifier in api/patterns.py. Ordered priority — first match wins.

  1. SHORT SQUEEZE DYNAMICS  (risk: extreme, color: red)
     Conditions: 3+ of:
     • compound >= +0.30 (extreme positive)
     • velocity accelerating AND change_percent >= 40%
     • sentiment_imbalance >= 10 pts (one-sided narrative)
     • hype_score >= 60

  2. MACRO FEAR PATTERN  (risk: high, color: light red)
     Conditions: 2+ of:
     • compound <= -0.15
     • negative article ratio >= 35%
     • sustained coverage with hype_score >= 30

  3. THEME / SECTOR WAVE  (risk: high, color: amber)
     Conditions: 3+ of:
     • compound >= +0.15
     • velocity accelerating
     • wide source distribution (source_conc <= 10) + hype >= 30
     • hype_score >= 40

  4. NARRATIVE COOL-OFF  (risk: moderate, color: blue)
     Conditions: 2+ of:
     • velocity decelerating
     • previously elevated hype (>= 25)
     • coverage drop >= 20%

  5. PRE-CATALYST BUILD  (risk: moderate, color: purple)
     Conditions: 1+ of:
     • accelerating volume with neutral tone (|compound| < 0.2)
     • building anticipation (0.05 <= |compound| < 0.25 AND vel >= 20)

  6. BALANCED COVERAGE  (risk: low, color: green)
     Fallback — no unusual signals detected


────────────────────────────────────────────────────────────────────────────────
  10. HYPE SCORE FORMULA (0–100)
────────────────────────────────────────────────────────────────────────────────

  Three factors combined:

  Factor 1: Sentiment Imbalance (0–40 pts)
  ├── Count positive vs negative articles
  ├── dominant_ratio = max(pos, neg) / total
  └── score = max(0, (dominant_ratio - 0.5) * 2) * 40
       → 0 pts if 50/50 split, 40 pts if 100% one direction

  Factor 2: Velocity Factor (0–35 pts)
  └── score = min(velocity_score / 100, 1.0) * 35

  Factor 3: Source Concentration (0–25 pts)
  ├── diversity = unique_sources / total_articles
  └── score = max(0, 1 - diversity) * 25
       → 0 pts if all different sources, 25 pts if all same source

  Level thresholds:
  ├── score >= 70  → extreme
  ├── score >= 45  → high
  ├── score >= 20  → moderate
  └── score <  20  → low


────────────────────────────────────────────────────────────────────────────────
  11. NARRATIVE DRIFT (api/drift.py)
────────────────────────────────────────────────────────────────────────────────

  Purpose: Show how sentiment has evolved over time (7d / 30d / 90d).

  Computation:
  1. Pull all SentimentScore records for last 91 days
  2. Aggregate by calendar day → {date, avg_compound, count, label}
  3. Window stats (7d/30d/90d):
     ├── avg_compound across the window
     ├── direction: positive/negative/neutral
     └── drift_direction: compare first half vs second half
          • delta > +0.05 → improving
          • delta < -0.05 → worsening
          • else          → stable

  Shift detection:
  1. Compute 3-day rolling average across timeline
  2. Walk backward — find last day where rolling avg crossed sentiment boundary
  3. Return: {detected, date, from_label, to_label, description}

  Frontend display:
  ├── Recharts AreaChart — color adapts to avg sentiment (green/red/gray)
  ├── Window selector: 7D / 30D / 90D
  ├── WindowStat pills showing avg_compound + drift direction icon
  └── Shift annotation box (purple accent when detected)

  12. AUTHENTICATION & API KEY MANAGEMENT
────────────────────────────────────────────────────────────────────────────────

  Auth: JWT (SimpleJWT)
  ├── POST /users/login/   → access_token (short-lived) + refresh_token
  ├── Access token stored in-memory (AuthContext)
  └── Refresh token stored in localStorage

  API Key encryption (users/encryption.py):
  ├── Fernet symmetric encryption (cryptography library)
  ├── Key derived from Django SECRET_KEY
  ├── User can save Gemini + Groq keys on Settings page
  └── Keys decrypted only at request time in market_chat view

  AI Chat flow:
  1. User posts question to /api/market/chat/<ticker>/
  2. Backend fetches user's encrypted keys from DB
  3. Decrypts on-the-fly, picks Gemini first (Groq as fallback)
  4. Builds context: ticker, company name, recent narratives
  5. Returns AI answer

  13. BACKGROUND SCRAPING

  15. ML MODEL (EXPERIMENTAL)
────────────────────────────────────────────────────────────────────────────────

  Goal: predict whether a narrative will move price UP / DOWN / FLAT.

  Features extracted per narrative (api/ml/features.py):
  ├── Sentiment scores (compound, positive, negative, neutral)
  ├── Source type encoding
  ├── Hour of day published
  ├── Days since last major narrative
  └── Current hype level encoding

  Training (manage.py train_model):
  ├── Pulls PriceImpact records with direction_24h label
  ├── RandomForestClassifier (scikit-learn)
  └── Saves to ml_models/price_impact_model.pkl

  Backtesting (manage.py backtest):
  ├── --model vader|finbert|both
  ├── Pearson correlation: sentiment vs next-day return
  ├── Directional accuracy
  └── Per-sentiment-label breakdown

  NOTE: This is experimental and does not drive the core product.
  The product's value is in narrative dynamics, not price prediction.

  16. HOW TO RUN LOCALLY
────────────────────────────────────────────────────────────────────────────────

  BACKEND
  ──────────────────────────────────
  cd backend
  source venv/bin/activate
  python3 manage.py migrate          # first time only
  python3 manage.py runserver        # runs on localhost:8000

  Required env vars (backend/.env):
  ├── SECRET_KEY=<django secret>
  ├── GNEWS_API_KEY=<your gnews key>
  ├── TWELVEDATA_API_KEY=<your key>
  └── REDIS_URL=redis://localhost:6379  (for Celery)

  FRONTEND
  ──────────────────────────────────
  cd frontend
  npm install                        # first time only
  npm run dev                        # runs on localhost:5173
  npm run build                      # production build

  Required env (frontend/.env):
  └── VITE_GNEWS_API_KEY=<your gnews key>  (legacy, now used by backend)

  CELERY (background scrapers)
  ──────────────────────────────────
  redis-server                       # start Redis
  cd backend && source venv/bin/activate
  celery -A config worker -l info    # worker
  celery -A config beat   -l info    # scheduler (triggers every 30 min)

  USEFUL MANAGEMENT COMMANDS
  ──────────────────────────────────
  python3 manage.py scrape_narratives --ticker AAPL
  python3 manage.py dataset_stats
  python3 manage.py backtest --model both
  python3 manage.py backtest_events --mode future
  python3 manage.py train_model
  python3 manage.py backfill_prices


────────────────────────────────────────────────────────────────────────────────
  17. CACHING STRATEGY
────────────────────────────────────────────────────────────────────────────────

  All expensive endpoints are cached using Django's cache framework.
  In dev: in-memory (LocMem). In prod: Redis.

  Cache key format    │ TTL    │ Endpoint
  ────────────────────┼────────┼─────────────────────────
  quote:<TICKER>      │ 10 min │ market/quote/
  chart:<TICK>:<RNG>  │ 10 min │ market/chart/
  search:<query>      │ 30 min │ market/search/
  news:<TICKER>       │ 15 min │ market/news/
  analyze:<TICKER>    │ 15 min │ market/analyze/
  drift:<TICKER>      │ 30 min │ market/drift/
  trending:top        │ 10 min │ market/trending/

  19. KNOWN LIMITATIONS & FUTURE WORK
────────────────────────────────────────────────────────────────────────────────

  Limitations
  ├── GNews free tier: 100 req/day — limits how many tickers can be tracked
  ├── SQLite in dev: no DISTINCT ON, some query patterns limited
  ├── FinBERT first load is slow (~5s) — model stays in memory after
  ├── ML price-prediction model is experimental, not production-ready
  └── No real-time WebSocket updates — polling only

  Potential next features
  ├── Watchlist: save tickers, get notified when pattern changes
  ├── Pattern change alerts (email/push when archetype shifts)
  ├── Sector-level narrative view (aggregate across sector)
  ├── Historical pattern replay ("what was the story 3 months ago?")
  ├── Backtest validation: does hype score predict reversals?
  ├── Premium tier: real-time Celery scraping for your watchlist
  └── Mobile app (React Native reusing the same backend)
